# 03. ページ設計

## 共通レイアウト

- ヘッダー: サイトロゴ + ナビゲーション（イベント / 登壇者 / 企画 / About）+ 「参加する」ボタン（PC）
- メインコンテンツ: ページ固有
- フッター: 「コミュニティ」「チャンネル登録」ボタン + コピーライト
- OGP / Twitter Card メタタグ対応

### モバイル対応

- ヘッダー: 「参加する」ボタン非表示（トップページのヒーローセクションに配置）、ナビリンク折り返し防止
- フッター: コピーライト中央寄せ

## 1. トップページ（/）

- ヒーローセクション:
  - コミュニティ名 + キャッチコピー
  - 統計カード（開催スケジュール / イベント開催数 / LT発表数）
  - モバイル: 開催スケジュールのみ表示 + 「参加する」ボタン
- イベント一覧:
  - カード形式、新着順
  - カード内容: イベント番号、タイトル、開催日、発表数、発表プレビュー
  - 開催予定イベントは青背景・左ボーダー・「開催予定」バッジ付き
  - デフォルト表示: 現在年のイベントのみ
  - 年度フィルター（年度ボタン + 「すべて」を末尾に配置）
  - 「すべて」選択時: 12件ずつの「さらに読み込む」ページネーション
  - 年フィルター選択時: 下部に前年・翌年ナビゲーションボタン（両サイド配置）

## 2. イベント詳細ページ（/events/NNN）

- パンくずリスト（イベント一覧 → 当該イベント）
- イベントヘッダー: タイトル、開催日、connpassリンク、ハッシュタグ
- イベントレポートリンク（report_url がある場合）
- YouTube動画:
  - サムネイル表示、クリックでiframe埋め込み再生（autoplay）
  - ローカル（file://）ではYouTubeページへのリンク動作（プログレッシブエンハンスメント）
- 発表一覧:
  - 登壇者情報（アイコン、名前、SNSリンク）
  - 発表タイトル
  - スライド埋め込み or リンク
  - 関連記事リンク
  - タグ表示
  - YouTube内タイムスタンプリンク
- ページ下部ナビゲーション: 左に前イベント、中央にイベント一覧、右に次イベント

## 3. 登壇者一覧ページ（/speakers）

- 検索ボックス（名前、ID、X、GitHub、発表タイトルで検索可能）
- ソート切替（登壇回数順 / 最新イベント順 / 名前順）
- 登壇者カード:
  - アイコン、名前、SNSリンク、登壇回数
  - 発表履歴: 最新3件を表示、4件以上は折りたたみ（展開/閉じるトグル）
- 20件ずつの「さらに読み込む」ページネーション（検索・ソートと連動）

## 4. 企画コーナーページ（/challenge）

- ページ見出し + リード文
- 2セクション構成:
  - 💪 チャレンジコーナー: `speaker.id == "easy2"` かつタイトルに「チャレンジコーナー」を含む発表
  - 🔍 ちょこっと深堀り！: `speaker.id == "easy2"` かつ上記以外の発表
- 各セクションにリード文（コーナー説明）
- カード形式: イベント番号、開催日、タイトル、動画タイムスタンプリンク
- カードクリックでイベント詳細ページへ

## 5. コミュニティ概要ページ（/about）

- コミュニティ紹介文 + Docswell スライド埋め込み
- 統計情報（イベント回数、LT発表数、登壇者数、connpassメンバー数）
- 運営メンバー（アイコン、名前、役割、SNSリンク）
- 参加方法（connpass / YouTubeアーカイブ）
- 関連リンク

## 埋め込み対応

- **YouTube**: サムネイル + クリックでiframe埋め込み再生（youtube-nocookie.com、autoplay）
- **スライド**: iframe埋め込み（Speaker Deck, Google Slides, Docswell等、自動URL生成）
- **記事**: リンク形式で表示
