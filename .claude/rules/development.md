# 開発ルール

## コーディング規約
- Python 3.9+ 互換のコードを書く
- 型ヒントを活用する
- docstring は Google スタイルで記述する

## Git ワークフロー

### ブランチ運用
- 修正・機能追加時は以下の流れで作業する:
  1. GitHub Issue を作成して作業内容を記録
  2. `main` から作業ブランチを切る
  3. ブランチ名は `feature/<概要>`、`fix/<概要>`、`docs/<概要>` 等のプレフィックス付き
  4. 作業ブランチで実装・コミット
  5. Pull Request を作成（関連 Issue をリンク）
  6. レビュー・確認後に `main` へマージ
- ただしユーザーから明示的に指示があった場合は `main` への直接プッシュも可

### コミット
- コミットメッセージは日本語 OK
- 機能単位で小さくコミットする

## テンプレート
- Jinja2 テンプレートは `src/templates/` に配置
- テンプレート内のコメントは HTML コメント形式で

## データ管理
- イベントデータは `data/events/NNN.yaml` 形式
- NNN はゼロ埋め3桁のイベント番号（例: 001, 076）
- YAML の文字コードは UTF-8
